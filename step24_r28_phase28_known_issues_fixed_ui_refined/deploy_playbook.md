# 部署 / 維修 / 回滾流程

## 1. 維修模式
建議保留：
- 正式版：`index_live.html`
- 維修頁：`index.html`

當要維修時：
1. 備份正式入口
2. 將維修頁部署成 `index.html`
3. 在本地 / 測試版處理新版本
4. 驗證完成後，再把正式版部署回 `index.html`

## 2. 測試版
建議保留一個測試入口，例如：
- `index_test.html`

這樣：
- 學生與老師進正式網址 → 維修頁或正式版
- 你自己可以直接測 `index_test.html`

## 3. 回滾
若新版有問題：
1. 立即把上一版穩定 `index.html` 部署回去
2. 新版繼續放在測試入口修補
3. 不要直接在正式入口上熱修多次

## 4. 部署前最後確認
- `js/config.js` 已填好
- Console 無紅字
- 老師登入成功
- 載入學生成功
- 老師加分成功
- voucher 正常
- AI 正常
- 學生核心流程正常
